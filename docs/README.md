## Di√°taxis Principles

### Tutorials

- **Purpose**: Teach beginners complete workflows
- **Approach**: Step-by-step, learning-oriented
- **Examples**: Setup guides, first-time configuration

### How-to Guides

- **Purpose**: Help users achieve specific goals
- **Approach**: Practical, problem-solution focused
- **Examples**: Feature implementation, troubleshooting, optimization

### Reference

- **Purpose**: Provide technical information
- **Approach**: Descriptive, comprehensive
- **Examples**: API docs, configuration options, specifications

### Explanation

- **Purpose**: Help users understand concepts and decisions
- **Approach**: Conceptual, discussion-oriented
- **Examples**: Architecture rationale, design decisions, methodology

## Contributing to Documentation
